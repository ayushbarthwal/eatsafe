# Food Quality Management System

The Food Quality Management System is a database-driven project designed to ensure effective tracking, evaluation, and maintenance of food quality across suppliers and production batches. Its primary objective is to automate and streamline the storage, retrieval, and analysis of food inspection data, reducing human errors and improving traceability in food safety operations.

## Features

- **Automated Test Recording:**  
  Uses stored procedures for efficient and validated test data entry.

- **Performance Optimization:**  
  Implements indexes for faster query execution.

- **Backup and Reporting:**  
  Utilizes SQL views for generating reports and maintaining backups.

- **Transaction Management:**  
  Ensures data consistency with rollback and commit operations.

- **Aggregated Analytics:**  
  Provides category-wise quality summaries and other analytics.

## Core Entities

- `FoodItem`: Details of food products.
- `Supplier`: Information about food sources.
- `Inspector`: Records of inspection personnel.
- `Batch`: Tracks production batches.
- `QualityTest`: Stores test parameters and outcomes.

## Technologies Used

- **Oracle SQL & PL/SQL:**  
  Advanced DBMS concepts including stored procedures, functions, transactions, indexing, and views.

## Project Structure

- SQL scripts for schema, procedures, functions, and views.
- Sample datasets for testing and demonstration.
- Documentation for setup and usage.
